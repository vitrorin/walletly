```bash

npx expo start          # Start dev server (scan QR with Expo Go on device)
npx expo start --android
npx expo start --ios

npx tsc --noEmit        # Type-check the mobile app

cd functions

```

## Architecture

**Stack:** Expo 54 (React Native + Web) + TypeScript + Firebase Auth + Firebase Firestore + Firebase Cloud Functions (v2, Node 20) + Claude API (Haiku)

**App name:** Walletly — a debit/credit card spending tracker. Users import bank transaction CSVs, the app auto-categorizes them, and Claude generates a weekly digest with saving recommendations.

**Data flow:** All state lives in Firestore under `users/{uid}/`. Firebase Auth (email + Google) gates all access. CSV files are uploaded to a Cloud Function for parsing and categorization; the client writes confirmed transactions directly to Firestore. Weekly digests are generated by a scheduled Cloud Function that calls Claude.

**Navigation:**
- Web: 160px left sidebar (Home, Transactions, Import, Digest + account at bottom)
- Mobile: bottom tab bar with the same 4 tabs
- Implemented via `Tab.Navigator` with a platform-conditional `tabBar` prop

**Firestore schema (`users/{uid}/`):**
```
cards/{cardId}:        { name, type, bank, lastFour, color }
transactions/{txId}:   { cardId, date, amount, merchant, category, rawDescription, excluded }
digests/{weekId}:      { weekStart, weekEnd, generatedAt, totalSpend, byCategory, summary, recommendations[] }
importHashes: string[] // on the user doc — prevents re-importing same CSV
```

**Cloud Functions (v2 API — `firebase-functions/v2/https`):**
- `parseCSV` — callable: receives CSV text + cardId, deduplicates via SHA-256 hash, parses + categorizes rows using keyword matching, returns transactions array for preview. Does NOT write to Firestore.
- `generateDigest` — callable: aggregates weekly transactions by category (no raw merchant names sent to Claude), calls Claude Haiku, writes digest to Firestore. Also callable by client for first-import trigger.
- `weeklyDigestScheduler` — scheduled (Monday 13:00 UTC): calls `runDigest()` for all users with transactions.

**Claude API key** is stored as a Firebase Secret (`CLAUDE_KEY` via `defineSecret`), never in source code. Access: `claudeKey.value()` inside the function.

**Category detection** uses keyword matching on raw merchant descriptions (in `functions/src/categoryKeywords.ts`). Claude is NOT used for categorization — only for the weekly digest summary.

**Key files:**
- `firebase.ts` — Firebase init; exports `db` and `auth`
- `src/types.ts` — `Card`, `Transaction`, `Digest`, `Category` types
- `src/navigation.ts` — `RootTabParamList`
- `src/utils/week.ts` — `getISOWeekId()`, `getWeekBounds()` helpers
- `src/navigation/AppNavigator.tsx` — platform-conditional Tab.Navigator
- `src/components/Sidebar.tsx` — web-only left nav
- `functions/src/types.ts` — server-side types (mirrors src/types.ts)
- `functions/src/categoryKeywords.ts` — keyword map + `categorize()`
- `functions/src/csvParser.ts` — pure CSV parsing logic (testable, no Firebase deps)
- `functions/src/generateDigest.ts` — exports `aggregateTransactions`, `buildClaudePrompt`, `runDigest`, and `generateDigest` callable

**Screens:**
- `HomeScreen` — card carousel, this-week stats, category breakdown bars, digest preview
- `TransactionsScreen` — full list, filter by card/category, tap to toggle `excluded`
- `ImportScreen` — card selector → file picker → Cloud Function call → preview table → confirm to Firestore
- `DigestScreen` — week navigator (◀ / ▶), AI summary, stats, recommendation cards

**`react-native-get-random-values` must be imported first** in `App.tsx` to polyfill `crypto.getRandomValues` for `uuid`.

## Deployment notes

- Cloud Functions require Firebase **Blaze** plan
- Set Claude API key: `firebase functions:secrets:set CLAUDE_KEY`
- Enable Firebase Auth in console: Email/Password + Google sign-in methods
- Firestore security rules: all `users/{uid}/**` requires `request.auth.uid == uid`
